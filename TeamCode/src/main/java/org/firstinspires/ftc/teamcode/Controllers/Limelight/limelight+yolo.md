## limelight加载yolov8n模型开发方案

### 一、模型选择

- **主检测模型**：YOLOv8n INT8 量化版（`.tflite`），输入分辨率 416×416，帧率 18–22 FPS
- **辅助校正模型（可选）**：YOLOv8n FP32 版（`.tflite`），每 30–50 帧运行一次，用于修正 INT8 的累积偏差
- **部署硬件**：Limelight 3A（四核 Cortex-A72 1.5GHz，纯 CPU 推理）

### 二、开发方案

采用 **半监督迭代伪标签 + 混合精度推理** 的完整流程：

1. **数据采集**：拍摄 1000 张含纹理彩色小球的图像
2. **初始标注**：人工标注其中 100 张
3. **FP32 模型训练**：用 100 张标注数据训练 YOLOv8n FP32 模型
4. **迭代生成伪标签**：
   - 用 FP32 模型对未标注图像推理，保留高置信度（>0.7）检测结果
   - 合并人工标签与伪标签，重新训练 FP32 模型
   - 重复 2–3 轮，直至获得约 900 张高质量伪标签
5. **INT8 模型训练**：使用最终完整数据集训练 INT8 量化模型（校准集 100–200 张）
6. **混合推理部署（可选）**：在 Limelight 3A 上通过python接口同时加载 INT8（主）和 FP32（辅），每 30 帧用 FP32 校正一次边界框偏移与置信度缩放

### 三、代码结构（Limelight Python SnapScript）

```
mixed_pipeline.py
├── 模型加载：interp_int8, interp_fp32
├── 预处理函数：preprocess()  → resize + 归一化
├── 后处理函数：postprocess() → NMS，返回检测列表
├── 校正函数：
│   ├── apply_correction()   → 应用当前偏差（dx,dy,dw,dh, scale）
│   └── compute_correction_from_fp32() → 更新移动平均偏差
├── 主处理函数 process_image()：
│   ├── 每帧运行 INT8 推理 → 得到 int8_dets
│   ├── 每隔 N 帧额外运行 FP32 推理 → 计算偏差并更新
│   └── 应用校正后输出到 NetworkTables
└── Pipeline 类封装
```

FTC 主控端（Java）通过 `NetworkTables` 读取 `limelight/detections` 字符串，格式：

```
class,conf,x1,y1,x2,y2;class,conf,...
```

### 四、预期效果

| 指标                                                  | 纯 INT8      | 混合推理（INT8+FP32校正） |
| --------------------------------------------------- | ----------- | ----------------- |
| **帧率**                                              | 18–22 FPS   | 17–20 FPS         |
| **精度**                                              | 中等（偶有偏移/误检） | 高（经 FP32 实时校正）    |
| **CPU 占用**                                          | 70–80%      | 80–90%            |
| **人工标注量**                                           | 1000 张      | **仅 100 张**       |
| **开发周期**                                            | 2–3 周       | 1.5–2 周           |
| 最终系统能在 FTC 比赛中稳定识别带不规则纹理的彩色小球，延迟 ≤60ms，满足实时跟踪与控制需求。 | <br />      | <br />            |
